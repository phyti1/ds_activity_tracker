In Folge werden die einzelnen Dateien heruntergeladen und in eine temporre CSV Datei test.csv geschrieben.

Nach dem Einlesen der temporren CSV-Datei, muss ein erstes Problem behandelt werden. Es existieren CSV Zeilen, welche die Dezimalstellen der Sensorwerte mit ',' anstatt '.' trennten. Da wir dieses Zeichen ebenfalls zur Trennung der CSV Eintrge nutzten, existieren nun Zeilen mit 33 Elemente anstatt 18. Das Problem knnte durch Spracheinstellungen oder Einstellungen des Smartphone Herstellers verursacht worden sein. Da nur wenige Werte mit 32 Elementen existieren, werden diese ignoriert.

Das Problem wurde bei Daten des Android Gertes von Flavio festgestellt. Um es zu beheben, werden die betroffenen Kommas in folgendem Code durch Punkte ersetzt.

In [ ]:
# convert 32 to 18 length
def bad_line_handler(bad_line):
    elements = bad_line[:3]
    for i in range(0, len(bad_line[3:]), 2):
        elements.append(f"{bad_line[i+3]}.{bad_line[i+4]}")
    return elements

lines_ignored = 0
lines_new = []

# convert 33 elements to 18, drop 32
for i, line in enumerate(lines):
    elements = lines[i].split(",")
    if(len(elements) == 18):
        lines_new.append(line)
    elif(len(elements) == 33):
        correct_line = bad_line_handler(elements)
        lines_new.append(str.join(",", correct_line))
    elif(len(elements) == 32):
        # other invalid format, ignore line
        lines_ignored += 1
        pass
    else:
        raise (f"ERROR: unknown error at line {i}")

print(f"{lines_ignored} Zeilen wurden ignoriert.")
print(f"{len(lines_new)} Zeilen sind nun verfgbar.")

In dieser Darstellung sind die gemessenen Sensor Readings pro Aktivitt je Person dargestellt. Im Challangekonzept wurde definiert, dass jede Person mindestens 20 Minuten jeder Aktivitt aufzuzeichnen soll. Diese Grenze ist als grne horizontale Linie eingezeichnet und diente uns zur berprfung, ob gengend Daten gesammelt wurden.

Die Anforderung konnte in der Aktivitt "Elevatoring" von drei Personen nicht erfllt werden. Im Lift war hufig das Senden der Daten nicht mglich, da keine Internetverbindung bestand. Da das autmatische Resending der App nicht wirklich funktionierte, wurden die Daten hufig nicht gepseichert. Wir waren uns dieser Einschrnkung bewusst und definierten mit den vorhandenen Daten weiterzuarbeiten.
